# Bone Cycle 开发方案

## 项目概述

火柴人自行车横版闯关游戏，过山车式的骑行体验。玩家控制火柴人骑着自行车在一条线上前进，遇到障碍物（如坑洞）需要跳跃通过。支持前进、倒车、停止控制，游戏开始前可选择火柴人和自行车的颜色。

## 核心玩法

### 1. 游戏机制
- **横向卷轴** - 画面从左向右跟随玩家
- **重力系统** - 车子会下落，需要在平台上骑行
- **速度控制** - D前进、A倒车、S停止，有惯性效果
- **跳跃机制** - 遇到坑洞时按空格跳跃

### 2. 关卡设计
- **第一关示例** - 平缓起步 → 小坑 → 连续坑 → 高台 → 终点
- **地形元素** - 平地、斜坡、坑洞、高台
- **收集物** - 金币、星星（可选，增加趣味性）

### 3. 角色系统
- **颜色选择** - 开始前选择火柴人和自行车颜色
- **颜色选项** - 火柴人（白/金/红）、自行车（蓝/绿/橙）
- **基础属性** - 生命值、速度、跳跃高度

## 技术选型

### 1. 前端技术
- **渲染引擎** - HTML5 Canvas
- **语言** - 原生JavaScript (ES6+)
- **构建工具** - Vite (开发体验好，配置简单)
- **无需框架** - 保持轻量，Canvas游戏不需要React/Vue

### 2. 物理系统
- **自研简易物理** - 只需要重力和碰撞检测
- **不需要专业引擎** - 游戏机制简单，不需要Box2D等

### 3. 资源管理
- **精灵图** - 火柴人动画可以用简单线条绘制
- **音频** - 后期添加背景音乐和音效

## 操作方案

### 玩家操作
- **D键** - 前进（加速）
- **A键** - 倒车（减速/后退）
- **S键** - 停止（刹车）
- **空格** - 跳跃
- **R** - 重新开始
- **ESC** - 暂停

### 颜色选择（开始页面）
- 火柴人颜色：白、金、红 三种可选
- 自行车颜色：蓝、绿、橙 三种可选
- 开始闯关后颜色固定

### 操作流程
1. 启动游戏 → 显示开始页面
2. 选择火柴人颜色（点击或按键1/2/3）
3. 选择自行车颜色（点击或按键4/5/6）
4. 点击"开始游戏"按钮
5. 进入第一关，使用D/A/S控制移动，空格跳跃
6. 到达终点 → 显示胜利画面
7. 掉入坑洞 → 生命值-1，重置到最近平台
8. 生命值归零 → 显示失败画面，可重新开始

### 物理机制
- 有惯性系统，松开按键后会慢慢停下
- 跳跃时保持当前水平速度
- 掉入坑洞后重置到最近平台
- 前进最大速度：5px/frame
- 倒车最大速度：-3px/frame
- 加速度：0.5px/frame
- 跳跃高度：150px

## 第一关详细设计

### 地形布局
```
起点 → [平地500px] → [坑100px] → [平地400px] → [坑100px] → [平地300px] → [高台200px] → [坑100px] → [平地1200px] → 终点
```

### 难度曲线
1. **前500px** - 安全区，让玩家适应操作
2. **第一个坑** - 简单跳跃，熟悉跳跃手感
3. **连续坑** - 需要连续跳跃
4. **高台** - 需要跳跃上台
5. **终点区域** - 胜利！

### 视觉提示
- 坑洞边缘用红色标记
- 终点有明显标识
- 收集物在安全位置

## 测试要点

1. **跳跃手感** - 跳跃高度和距离是否合适
2. **碰撞检测** - 是否会卡住或穿模
3. **难度平衡** - 第一关是否太难或太简单
4. **性能表现** - 帧率是否稳定60fps
5. **键盘响应** - 操作是否灵敏
6. **颜色选择** - 开始页面颜色切换是否正常
7. **速度控制** - 前进/倒车/停止是否流畅

## 技术风险

1. **物理引擎** - 简单实现可能有bug，需要调试
2. **碰撞检测** - 边缘情况需要处理
3. **性能** - Canvas渲染大量对象可能卡顿
4. **惯性系统** - 速度衰减需要调参，避免太滑或太滞
5. **颜色选择** - 需要处理开始页面和游戏状态切换
